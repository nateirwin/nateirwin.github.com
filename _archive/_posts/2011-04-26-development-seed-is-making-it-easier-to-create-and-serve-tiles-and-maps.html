Interesting news today in the world of web mapping: Development Seed, a company that works primarily in the international development space, announced that they are releasing two new commercially-focused products: a hosted TileStream service and a TileMill/TileStream appliance that is mobile and can be deployed behind corporate firewalls. If you don’t know much about TileMill or TileStream, here’s a brief explanation: These two standalone but complementary open source software projects work together to allow cartographers to design, create, and serve image tiles for use in web maps. The hosted service and appliance will undoubtedly make it easier for someone out there to get web maps up and running, and that is a good thing.

Although these are interesting products in and of themselves, what makes TileMill and TileStream both interesting to me and relevant to my organization is their support for the MBTiles format and UTFGrid encoding scheme. These two implementations combined make up the “holy grail” of web mapping; using them together allows you to utilize tiled images to render data over web maps while still supporting interactive mouseover/click behavior that users have come to expect when using modern web maps.

MBTiles is a portable format that utilizes SQLite to store image tiles as blob data. A MBTiles file can be easily transferred over the network, and can serve tiles even when a mobile device is disconnected from the network. In short, MBTiles makes it easy to store, transfer, and serve image tiles for use in web mapping applications. The UTFGrid encoding scheme adds support for interactive mouse events to a set of tiles stored in the MBTiles format. In many cases, this eliminates the need for a call to the server to get information about a geospatial feature.

What’s funny is that Bill Thorp of the NPMap team developed a similar approach a few years back. We were looking at ways to improve performance, and wrote a layer handler for NPMap that built out an R-tree for a geospatial layer on the server, then pushed it out to the web browser where it could be used to dynamically “query” the R-tree generated on top of the ArcGIS Server service. Our implementation wasn’t quite as refined or elegant as Development Seed’s, though, so I really look forward to utilizing TileMill and TileStream in some of our NPMap projects.
