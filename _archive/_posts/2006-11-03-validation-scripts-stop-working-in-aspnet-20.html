--- layout: post title: Validation Scripts Stop Working in ASP.NET 2.0 published: true meta: _edit_last: "2" tags: - Development type: post status: publish --- Another problem and solution to a cryptic ASP.NET 2 problem. This is related to the issue with ASP.NET 1.1, ASP.NET 2.0, 64-bit, and IIS 6.0 that I discussed in my last blog post. I noticed that all of my client-side validation scripts stopped working all of a sudden. Well, this was a major problem, as I didn't have any code to catch errors. The validation scripts were stopping errors (such as required fields being left blank) on the client-side before they were posted to the server. I did a Google search and found that many others had experienced the same "mysterious" behavior. Well, I finally found a solution to my problem at ASPNetResources.com. Turns out that when ASP.NET 1.1 was installed the other day on our 64-bit machine it messed with the scripts that are stored in the aspnet_client directory of the application. So, the solution simply involved removing all client-side scripts (command line: c:\windows\microsoft.net\framework64\v2.0.50727\aspnet_regiis.exe -ea) and then re-installing them (command line: c\windows\microsoft.net\framework64\v2.0.50727\aspnet_regiis.exe -c). The thing is, the validation just stopped working today while the ASP.NET 1.1 issue was solved a couple of days ago. Humm... Listening to Brad Mehldau - Live in Tokyo...
